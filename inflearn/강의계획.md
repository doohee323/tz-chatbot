# tz-chatbot 기반 인프런 강의 계획

> 참조: `refer/tz-vibecoding` — 섹션별 본문 작성 → `PPT-슬라이드` 생성 순서로 진행

---

## 1. 강의 개요

### 1.1 강의 주제
**바로쓰는 RAG 챗봇 on K8S**

**부제**: RAG + Dify 챗봇을 Kubernetes에서 구축하고 운영하기

- **이 앱/환경은 전부 Kubernetes(K8s) 기반입니다.** RAG, Dify, chat-gateway, chat-admin, MinIO, Qdrant 등 모든 컴포넌트가 K8s 클러스터 위에서 동작하며, 강의에서도 K8s 기준으로 설치·설정·운영을 다룹니다.
- RAG(Retrieval-Augmented Generation)와 Dify를 활용한 챗봇 시스템을 **K8s 환경**에서 처음부터 설치·설정·운영하는 과정
- 실습 프로젝트: **tz-chatbot** — 단일 RAG·LangChain·Dify 데모를 넘어 **복수의 프로젝트(앱)에 챗봇을 적용**할 수 있도록 구성된 **K8s 기반** 플랫폼
  - **DB**: 시스템(앱) 등록·설정, 대화·메시지 저장·조회 (chat_admin)
  - **배치**: Dify → DB 동기화(/v1/sync), 토픽별 RAG Ingestion CronJob, 관리 화면에서 재색인 트리거
  - **관리 화면(chat-admin)**: 시스템(DrillQuiz, CoinTutor 등) CRUD, 채팅 조회, RAG 문서 업로드·재색인
  - chat-gateway가 여러 클라이언트 앱의 채팅을 한 곳에서 라우팅·관리

### 1.2 대상 수강생
- **Kubernetes(K8s) 기본 사용법**을 알고 있는 개발자/DevOps (이 강의는 K8s 기반 환경을 전제로 합니다)
- RAG·LLM·챗봇 구축에 관심 있는 개발자
- Dify 또는 유사 LLM 오케스트레이션 도구를 **K8s 위에서** 실무에 적용하려는 사람

### 1.3 학습 목표 (초안)
- **이 앱이 K8s 기반**임을 이해하고, RAG·Dify·게이트웨이·관리 화면이 모두 K8s 위에서 동작한다는 전제 하에 학습
- RAG의 개념(검색·임베딩·벡터 DB)과 Dify의 역할 이해
- **LangChain**의 역할(Loader, Splitter, Embedding, VectorStore)과 본 프로젝트에서의 사용처 이해
- Git 저장소 클론·환경설정·**로컬 테스트**로 실습 준비 완료
- MinIO, Qdrant, RAG Backend, Dify를 K8s에서 설치·연동
- 토픽별 RAG 분리(CoinTutor / DrillQuiz) 설계 및 적용
- **복수 프로젝트 챗봇 지원** 구조 이해: DB(시스템·대화 저장), 배치(동기화·Ingestion), 관리 화면(시스템 CRUD·RAG 업로드·재색인)
- chat-gateway·chat-admin으로 멀티 앱 채팅 게이트웨이 구성
- **서버 배포 방법**(CI/k8s, Jenkins) 및 **개발/운영 환경** 구분 가이드
- Ingress·TLS·시크릿 등 운영에 필요한 설정 정리
- **운영 시 시스템 확장(Scaling)**: HPA, replicas, metrics-server

### 1.4 예상 강의 시간
- **총 14~16시간** (LangChain·Git·로컬테스트·배포·스케일링 반영)

## 2. 작성 원칙 (tz-vibecoding 참고)

1. **섹션별 본문 먼저**: 각 섹션 폴더에 `01-제목.md`, `02-제목.md` 형태로 **개념 설명** 작성
2. **실습은 중간 삽입**: 개념 뒤에 `04-1-실습-개요.md`, `04-2-실습-환경준비.md` 형태로 **실습 강의** 추가
3. **PPT는 나중에**: 모든 섹션 본문이 준비된 뒤 `PPT-슬라이드/` 폴더에 섹션별 슬라이드 md 생성
4. **한 파일 = 한 강의(또는 한 슬라이드)**: 재사용·수정·PPT 매핑이 쉽도록 단위를 작게 유지

## 6. 작성 순서 제안

1. **1단계: 계획 확정**  
   - 본 계획 검토 후 섹션·목차·실습 위치 확정

2. **2단계: 섹션별 본문 작성**  
   - **섹션01 → 섹션02 → … → 섹션13** 순으로 진행  
   - 각 섹션 내: 개념 강의(01, 02, …) → 실습 강의(N-1, N-2, …) 순서로 작성  
   - 참고: 프로젝트 루트 `README.md`, `rag/README.md`, `docs/*.md`, `refer/tz-vibecoding` 본문 톤 유지

3. **3단계: PPT 슬라이드 생성**  
   - 모든 섹션 본문이 준비된 뒤  
   - `PPT-슬라이드/섹션NN-제목/` 아래에 `슬라이드-01-제목.md` 형태로 본문을 요약·슬라이드용으로 변환  
   - 참고: `refer/tz-vibecoding/PPT-슬라이드` 형식(제목, 슬라이드 내용, 발표 노트)

4. **4단계: 정리**  
   - `강의상세소개.md`, `강의학습목표.txt` 등 인프런 업로드용 메타 정리
