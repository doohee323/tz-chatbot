# 03. 개발/운영 환경 가이드

## 브랜치와 네임스페이스

- **main** → 운영: 네임스페이스 **devops** (또는 production용 NS), Secret 접미사 **-main**
- **qa** → QA: **devops** 또는 **devops-qa**, Secret **-qa**
- **dev** → 개발: **devops-dev**, Secret **-dev**
- README “Branches and namespaces” 표와 CI 스크립트·Jenkinsfile을 함께 확인하세요.

## 환경별 차이

- **로컬**: SQLite, localhost, .env 파일. Dify는 별도 URL(또는 mock)
- **개발(dev)**: devops-dev NS, 별도 Dify/ RAG 인스턴스 사용 가능
- **운영(main)**: 공용 Dify·RAG·DB, 실제 도메인·TLS

## 설정 관리

- **Secret**: 브랜치별로 다른 Secret 이름을 쓰면, 같은 매니페스트로 다른 값을 주입 가능
- **환경변수**: DIFY_BASE_URL, DIFY_*_API_KEY 등 앱별·환경별로 다르게 두기
- **도메인**: Ingress에서 호스트를 환경별로 나누거나, 경로로 나눌 수 있음
