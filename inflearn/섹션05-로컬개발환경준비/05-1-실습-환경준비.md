# 05-1. 실습: 환경 준비

## 목표

- tz-chatbot 저장소를 클론하고, 실습에 필요한 최소 환경을 준비합니다.

## 단계

### 1. 저장소 클론

```bash
git clone <저장소 URL>
cd tz-chatbot
```

- 사용할 브랜치로 체크아웃: `git checkout main` (또는 dev/qa 등)

### 2. KUBECONFIG 설정

```bash
export KUBECONFIG=~/.kube/config   # 또는 실제 사용하는 config 경로
kubectl get nodes
```

- 노드 목록이 보이면 클러스터 접근 준비 완료

### 3. (로컬 실행 시) chat-gateway 환경

```bash
cd chat-gateway
cp .env.example .env   # 있다면
# .env에 DIFY_BASE_URL, DIFY_API_KEY, CHAT_GATEWAY_JWT_SECRET 등 설정
```

- chat-admin도 동일하게 해당 디렉터리에서 `.env` 또는 환경변수 설정 (각 README 참고)

### 4. 확인

- `kubectl get ns` 로 네임스페이스 확인
- 다음 실습(05-2: 클러스터 접속 검증, 05-3: 로컬 테스트)으로 넘어갑니다.
