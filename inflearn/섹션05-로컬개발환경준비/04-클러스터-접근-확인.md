# 04. 클러스터 접근 확인

## 확인 방법

```bash
# 사용 중인 컨텍스트·클러스터
kubectl config current-context

# 노드 목록
kubectl get nodes

# 네임스페이스
kubectl get ns
```

- `get nodes`에서 노드가 Ready 상태로 보이면 해당 클러스터에 대한 권한이 있는 것입니다.
- 여러 클러스터를 쓰는 경우 `kubectl config use-context <context-name>`으로 전환 후 확인하세요.

## bootstrap 전 확인

- bootstrap.sh는 Phase 0에서 `kubectl get nodes`를 실행해 실패하면 중단합니다.
- 실습 전에 한 번 직접 실행해 두면, 이후 설치 단계에서 “클러스터 접속 불가” 오류를 줄일 수 있습니다.
