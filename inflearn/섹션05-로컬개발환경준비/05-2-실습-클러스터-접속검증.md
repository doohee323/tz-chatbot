# 05-2. 실습: 클러스터 접속 검증

## 목표

- 현재 사용 중인 KUBECONFIG로 Kubernetes 클러스터에 정상 접속되는지 검증합니다.

## 단계

### 1. 컨텍스트 확인

```bash
kubectl config current-context
```

- 실습에 사용할 클러스터의 컨텍스트인지 확인

### 2. 노드·네임스페이스 확인

```bash
kubectl get nodes
kubectl get ns
```

- 노드가 Ready 상태인지, 필요한 네임스페이스(default, devops, rag, dify 등)가 있는지 확인
- 아직 없으면 이후 bootstrap.sh 또는 수동으로 생성하면 됨

### 3. (선택) Helm 확인

```bash
helm list -A
# 또는
helm version
```

- RAG·Dify·Ingress 설치 시 Helm을 쓰므로, 사용할 계정에 Helm 접근 권한이 있는지 확인

### 4. 완료

- 위 명령들이 오류 없이 실행되면, 다음 섹션(06: RAG 스택 설치) 또는 05-3(로컬 테스트)으로 진행할 수 있습니다.
