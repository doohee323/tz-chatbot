# 03. 강의 구성

강의는 **13개 섹션**으로 구성되며, **K8s 기반** 환경에서 RAG·Dify·챗봇 게이트웨이를 설치·운영하는 순서로 진행합니다.

| 섹션 | 제목 | 예상 시간 |
|------|------|-----------|
| 01 | 강의 소개 | 0.5h |
| 02 | 실습 프로젝트 소개 (tz-chatbot) | 0.5h |
| 03 | RAG와 챗봇 아키텍처 개념 | 1h |
| 04 | Kubernetes 인프라 개요 | 0.5h |
| 05 | 로컬/개발 환경 준비 (Git·환경설정·로컬 테스트) | 1.5h |
| 06 | RAG 스택 설치 (install.sh) | 1.5h |
| 07 | 토픽별 RAG 분리 (CoinTutor / DrillQuiz) | 1.5h |
| 08 | RAG 문서 수집·임베딩·Ingestion | 1.5h |
| 09 | Dify 설치 및 RAG 도구 연동 | 1.5h |
| 10 | chat-gateway와 chat-admin (복수 프로젝트 지원) | 1.5h |
| 11 | Ingress·도메인·TLS | 1h |
| 12 | 운영·모니터링·스케일링 | 1.5h |
| 13 | 종합 정리 및 확장 | 0.5h |

**총 예상 시간**: 약 14~16시간

- 앞부분(01~04): **개념·전제** — K8s 기반, tz-chatbot 구조, RAG·Dify 개념, K8s 인프라
- 중간(05~09): **설치·연동** — 환경 준비, RAG 스택, 토픽 분리, Ingestion, Dify
- 뒤부분(10~13): **게이트웨이·운영** — chat-gateway·chat-admin, Ingress/TLS, 운영·스케일링, 정리

각 섹션에는 **개념 설명**과 필요 시 **실습**이 포함됩니다. 실습은 해당 섹션 폴더 내 `NN-N-실습-제목.md` 형태로 제공됩니다.
