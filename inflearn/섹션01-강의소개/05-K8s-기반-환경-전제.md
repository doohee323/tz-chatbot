# 05. K8s 기반 환경 전제

이 강의의 **전제 조건**을 정리합니다.

## 모든 컴포넌트가 K8s에서 동작한다

| 컴포넌트 | K8s 리소스 예 | 비고 |
|----------|----------------|------|
| Ingress | Ingress, cert-manager | TLS 라우팅 |
| MinIO | Deployment/StatefulSet, Service | `devops` NS, rag-docs 버킷 |
| Qdrant | Helm, PVC | `rag` NS |
| RAG Backend | Deployment, Service | 토픽별(CoinTutor, DrillQuiz) |
| RAG Ingestion | Job, CronJob | MinIO → 임베딩 → Qdrant |
| Dify | Helm | `dify` NS |
| chat-gateway | Deployment, Service | `devops` NS |
| chat-admin | Deployment, Service | `devops` NS |

로컬에서 **gateway.sh**, **admin.sh**로 실행하는 것은 “동작 확인·개발용”이며, 실서비스는 모두 K8s에 배포된다고 보면 됩니다.

## 필요한 도구·환경

- **kubectl**: 설치되어 있고, 클러스터에 접속 가능
- **KUBECONFIG**: 사용할 클러스터를 가리키도록 설정됨
- **(선택)** Helm: Dify·Qdrant 등 Helm 차트 설치 시 사용
- **(선택)** Git: 저장소 클론·브랜치 사용

## 클러스터 준비

- 강의 실습을 위해 **Kubernetes 클러스터가 이미 존재**하고, `kubectl get nodes` 등으로 접근 가능한 상태를 가정합니다.
- 클러스터를 직접 구축하는 방법은 이 강의 범위 밖입니다. (예: “바로 쓰는 CI/CD on K8S” 등 별도 강의 참고)

이 전제를 만족하면 섹션 05(환경 준비)부터 실습을 진행할 수 있습니다.
