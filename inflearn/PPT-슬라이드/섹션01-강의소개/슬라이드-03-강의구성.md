# 슬라이드 03: 강의 구성

## 슬라이드 내용 (한 장)

**13개 섹션, 총 약 14~16시간**

| 구간 | 섹션 | 내용 |
|------|------|------|
| **앞부분** (01~04) | 강의 소개, 실습 프로젝트 소개, RAG·챗봇 아키텍처, K8s 인프라 | 개념·전제 |
| **중간** (05~09) | 환경 준비, RAG 스택 설치, 토픽별 RAG 분리, Ingestion, Dify | 설치·연동 |
| **뒷부분** (10~13) | chat-gateway·chat-admin, Ingress·TLS, 운영·스케일링, 종합 정리 | 게이트웨이·운영 |

• 각 섹션: 개념 설명 + 필요 시 실습 (섹션 폴더 내 실습 md)
• 01 약 0.5h → 02~04 각 0.5~1h → 05~10 각 1~1.5h → 11~13 각 0.5~1.5h

---

## 발표 노트

강의는 총 13개 섹션으로 구성되어 있고, 예상 학습 시간은 14시간에서 16시간 정도입니다.

앞부분, 1번부터 4번 섹션까지는 강의 소개, 실습 프로젝트인 tz-chatbot 소개, RAG와 챗봇 아키텍처 개념, 그리고 Kubernetes 인프라 개요를 다룹니다. 여기서는 K8s 기반이라는 전제와 전체 구조를 이해하는 데 집중합니다.

중간 구간인 5번부터 9번까지는 로컬·개발 환경 준비, RAG 스택 설치, 토픽별 RAG 분리, 문서 수집·임베딩·Ingestion, Dify 설치와 RAG 도구 연동을 다룹니다. 실제로 설치하고 서로 연동하는 실습이 이어집니다.

뒷부분 10번부터 13번까지는 chat-gateway와 chat-admin으로 복수 앱을 지원하는 방법, Ingress와 도메인·TLS 설정, 운영과 모니터링·스케일링, 마지막으로 종합 정리와 확장 주제를 다룹니다. 게이트웨이와 관리 화면, 그리고 실제 운영에 필요한 설정과 스케일링까지 다룹니다.

각 섹션에는 개념 설명이 들어가고, 필요하면 해당 섹션 폴더 안에 실습용 마크다운이 있어서 단계별로 따라 할 수 있습니다. 시간 배분은 1번이 약 30분, 2번부터 4번까지 각각 30분에서 1시간, 5번부터 10번까지는 각 1시간에서 1시간 30분, 11번부터 13번까지는 30분에서 1시간 30분 정도로 생각하시면 됩니다.
