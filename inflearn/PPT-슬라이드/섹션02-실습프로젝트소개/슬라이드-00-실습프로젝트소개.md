# 슬라이드 00: 실습 프로젝트 소개

## 슬라이드 내용 (한 장)

**tz-chatbot — K8s 기반 RAG + Dify 챗봇 플랫폼**

• 단순 “RAG 하나 + Dify 하나” 데모가 아님  
• **여러 클라이언트 앱**(DrillQuiz, CoinTutor 등)에 **챗봇 적용** 가능  
• **DB**(시스템·대화 저장), **배치**(동기화·Ingestion), **관리 화면**(chat-admin) 보유

**K8s 기반 동작**  
• RAG, Dify, chat-gateway, chat-admin, MinIO, Qdrant, Ingress → 모두 K8s 클러스터 위 Pod·Service·Ingress  
• 실습: `kubectl`, `bootstrap.sh`, `install.sh` 로 설치·확인

**이 강의에서의 위치**  
• 실습 프로젝트 = 강의 전반의 설치·설정·운영 대상 저장소  
• 다음에서 플로우·컴포넌트, 복수 프로젝트 지원, 디렉터리, 배포 순서 상세

---

## 발표 노트

이번에는 실습 프로젝트인 tz-chatbot을 소개하겠습니다.

tz-chatbot은 K8s 기반에서 동작하는 RAG와 Dify 챗봇 플랫폼입니다. 단순히 RAG 하나, Dify 하나만 올리는 데모가 아니라, DrillQuiz나 CoinTutor 같은 여러 클라이언트 앱에 챗봇을 적용할 수 있도록 설계되어 있습니다. 그래서 시스템과 대화를 저장하는 DB, 동기화와 RAG Ingestion을 담당하는 배치, 그리고 chat-admin이라는 관리 화면까지 갖춘 구조입니다.

모든 컴포넌트가 Kubernetes 클러스터 위에서 Pod, Service, Ingress로 배포됩니다. 실습에서는 kubectl과 bootstrap.sh, install.sh 등을 사용해 이 환경을 설치하고 확인합니다.

이 강의에서 tz-chatbot은 실습 프로젝트, 즉 강의 전반에서 설치·설정·운영의 대상이 되는 저장소입니다. 다음 슬라이드부터는 요청 플로우와 컴포넌트, 복수 프로젝트 챗봇 지원이 어떻게 되어 있는지, 디렉터리 구조, 그리고 배포 순서와 의존 관계를 자세히 다룹니다.
