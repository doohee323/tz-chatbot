# 01. RAG란

## RAG (Retrieval-Augmented Generation)

- **의미**: “검색(Retrieval)으로 보강(Augmented)한 생성(Generation)”
- 사용자 **질의**가 들어오면, 미리 저장해 둔 **문서/지식**에서 관련 부분을 **검색**하고, 그 내용을 **LLM 생성**에 넣어 답변합니다.

## 왜 쓰나

- **LLM만** 쓰면: 학습 시점 이후 지식·내부 문서를 모르고, 환각(hallucination) 가능성도 있음
- **RAG**를 쓰면:
  - 회사 문서, 매뉴얼, FAQ 등 **최신·전문 지식**을 검색해 답에 반영
  - “이 문서에 있는 내용만 기준으로 답해라”처럼 **출처를 제한**해 환각을 줄일 수 있음

## 흐름 (간단)

1. **오프라인**: 문서를 잘게 나누고(청킹), 각 청크를 **임베딩**해 **벡터 DB**에 저장
2. **온라인**: 사용자 질의를 임베딩 → 벡터 DB에서 **유사 청크 검색** → 검색된 텍스트를 LLM **컨텍스트**에 넣고 답변 생성

tz-chatbot에서는 **MinIO(문서)** → **Ingestion(청킹·임베딩)** → **Qdrant(벡터 DB)** → **RAG Backend(검색 API)** → **Dify(LLM·도구 호출)** 순으로 이 흐름이 구현됩니다.
