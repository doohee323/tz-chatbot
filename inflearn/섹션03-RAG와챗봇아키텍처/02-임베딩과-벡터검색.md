# 02. 임베딩과 벡터 검색

## 임베딩 (Embedding)

- 텍스트(문장·문단)를 **고정 차원의 벡터(숫자 배열)** 로 바꾸는 과정
- “의미가 비슷한 텍스트는 벡터도 가깝게” 되도록 학습된 **임베딩 모델** 사용 (OpenAI, Gemini 등)
- 본 프로젝트: Ingestion·RAG Backend에서 **OpenAI** 또는 **Google Gemini** 임베딩 사용 (환경변수로 선택)

## 벡터 검색

- 사용자 **질의**도 같은 임베딩 모델로 벡터화
- 벡터 DB에서 **질의 벡터와 가장 가까운** 벡터들(코사인 유사도 등)을 찾음 → 해당하는 **문서 청크**를 가져옴
- 그 청크들을 LLM에게 “참고 자료”로 넘겨 답변 생성

## 청킹 (Chunking)

- 긴 문서를 **일정 크기 단위(청크)** 로 자르는 것
- 너무 크면 검색 정확도·토큰 제한에 불리, 너무 작으면 문맥이 깨짐
- tz-chatbot: **LangChain RecursiveCharacterTextSplitter** 사용 (chunk_size, chunk_overlap 등 환경변수로 조정). 섹션 08에서 다룸.
