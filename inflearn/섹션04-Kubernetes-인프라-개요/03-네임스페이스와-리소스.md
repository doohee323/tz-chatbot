# 03. 네임스페이스와 리소스

## 사용하는 네임스페이스

| 네임스페이스 | 주요 리소스 |
|--------------|-------------|
| **default** | Ingress NGINX (Controller, cert-manager 관련) |
| **devops** | MinIO, chat-gateway, chat-admin (배포 시), PostgreSQL(별도 설치 시) |
| **rag** | Qdrant, RAG Backend(Deployment), RAG Ingestion(Job/CronJob), ConfigMap/Secret |
| **dify** | Dify Helm 릴리스 (API, Worker 등) |

## 리소스 종류 (요약)

- **Deployment / StatefulSet**: Pod 템플릿·replica 수. RAG Backend, Dify, chat-gateway, chat-admin 등
- **Service**: Pod 그룹에 대한 내부/외부 접근 (ClusterIP, LoadBalancer 등)
- **Ingress**: HTTP/HTTPS 라우팅 규칙 (호스트·경로 → Service)
- **ConfigMap / Secret**: 설정·비밀값 (Ingestion용 MinIO/Qdrant/API Key 등)
- **Job / CronJob**: RAG Ingestion 한 번 실행 또는 주기 실행

브랜치별로 **devops** / **devops-dev** / **devops-qa** 처럼 네임스페이스를 나누어 쓰는 경우는 섹션 12(개발/운영 환경 가이드)에서 다룹니다.
